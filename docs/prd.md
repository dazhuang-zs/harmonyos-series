# 产品需求文档（PRD）
## HarmonyOS 开发助手（MiMo 驱动）

---

## 一、产品概述

### 1.1 产品定位

**HarmonyOS 开发助手**是一个基于小米 MiMo 大模型驱动的智能开发辅助工具，旨在降低鸿蒙应用开发门槛，提升开发者效率。

**核心价值**：
- 📚 **知识获取**：RAG 检索官方文档 + CSDN 优质文章，秒级回答开发问题
- 💻 **代码生成**：自然语言 → ArkTS/ArkUI 可运行代码
- 🔧 **报错诊断**：粘贴报错 → 自动定位原因 + 给出修复方案
- 🔄 **代码迁移**：Android → ArkTS 自动转换
- 📄 **内容生产**：自动生成技术文章并发布到 CSDN

### 1.2 目标用户

| 用户类型 | 特征 | 痛点 |
|---------|------|------|
| 鸿蒙初学者 | 刚接触 ArkTS，不熟悉语法 | 学习曲线陡峭，文档难找 |
| Android 开发者 | 需要迁移现有 App 到鸿蒙 | 代码迁移成本高，语法差异大 |
| 全职鸿蒙开发者 | 日常开发，追求效率 | 重复代码编写耗时，报错调试麻烦 |
| 技术内容创作者 | 写鸿蒙教程、技术文章 | 整理素材耗时，发布流程繁琐 |

### 1.3 产品边界

**包含范围**：
- ✅ RAG 文档问答
- ✅ ArkTS 代码生成
- ✅ 报错诊断
- ✅ Android → ArkTS 代码迁移（基础功能）
- ✅ CSDN 草稿箱自动发布

**不包含范围**（V1.0）：
- ❌ 完整 IDE（如 DevEco Studio 替代品）
- ❌ 鸿蒙系统底层开发（驱动、内核）
- ❌ 多语言支持（V1.0 仅支持中文，V2.0 增加英文）

## 二、功能需求

### 2.1 功能模块总览

```
HarmonyOS 开发助手
├── Web UI（Next.js）
│   ├── 问答页面（RAG Q&A）
│   ├── 代码生成页面
│   ├── 报错诊断页面
│   ├── 代码迁移页面
│   └── CSDN 发布管理页面
├── VS Code 插件（可选）
├── 命令行工具（CLI）
└── REST API（FastAPI）
```

### 2.2 功能详情

#### 功能 1：RAG 文档问答

**用户故事**：
> 作为鸿蒙初学者，我想要快速找到 ArkTS 语法的正确答案，而不是在官方文档里翻半天。

**功能描述**：
- 用户输入自然语言问题（如"@State 和 @Prop 有什么区别？"）
- 系统检索华为官方文档 + CSDN 优质文章
- 返回准确答案（含代码示例、官方文档链接）

**输入**：
- 问题文本（必填）
- 上下文会话 ID（可选，用于多轮对话）

**输出**：
- 答案（Markdown 格式）
- 参考来源（官方文档链接、CSDN 文章链接）
- 相关代码示例

**验收标准**：
- 答案准确率 ≥ 85%（人工评估 100 个常见问题）
- 响应时间 < 5 秒（P95）
- 支持多轮对话（记住上下文）

---

#### 功能 2：ArkTS 代码生成

**用户故事**：
> 作为鸿蒙开发者，我想要用自然语言描述需求，直接生成可运行的 ArkTS 代码，减少重复编码。

**功能描述**：
- 用户输入自然语言需求（如"创建一个带搜索栏的列表页"）
- 系统生成完整的 ArkTS/ArkUI 代码
- 可选：生成对应的单元测试代码

**输入**：
- 需求描述（必填）
- 代码风格偏好（可选：函数式 / 面向对象）
- 是否生成单元测试（可选，默认否）

**输出**：
- 完整代码文件（.ets）
- 代码说明（注释）
- 依赖说明（需要导入哪些包）

**验收标准**：
- 代码可运行率 ≥ 90%（编译无错误）
- 代码质量得分 ≥ 7/10（人工评估可读性、规范性）
- 支持生成完整 DevEco Studio 项目骨架

---

#### 功能 3：报错诊断

**用户故事**：
> 作为鸿蒙开发者，我想要粘贴编译错误，系统自动告诉我哪里错了、怎么修，而不是盯着报错日志发呆。

**功能描述**：
- 用户粘贴编译错误日志
- 系统分析错误类型和可能原因
- 返回分步修复方案（含代码示例）

**输入**：
- 错误日志文本（必填）
- 相关代码片段（可选，有助于更精准诊断）

**输出**：
- 错误类型（如"模块未找到"、"类型不匹配"）
- 可能原因（Top 3）
- 分步修复方案（含代码示例）
- 相关文档链接

**验收标准**：
- 错误识别准确率 ≥ 80%
- 修复方案可用率 ≥ 70%（按方案操作后能解决问题）
- 响应时间 < 10 秒（P95）

---

#### 功能 4：Android → ArkTS 代码迁移

**用户故事**：
> 作为 Android 开发者，我想要把现有的 Android App 迁移到鸿蒙，但不知道怎么把 Java/Kotlin 代码转换成 ArkTS。

**功能描述**：
- 用户粘贴 Android Java/Kotlin 代码
- 系统生成等价的 ArkTS 实现
- 标注迁移注意事项（如 API 差异）

**输入**：
- Android 代码（必填，支持 Java/Kotlin）
- 目标鸿蒙版本（可选，默认 HarmonyOS 5.0）

**输出**：
- 等价 ArkTS 代码
- 迁移注释（标注 API 差异、注意事项）
- 对比视图（Before: Android / After: ArkTS）

**验收标准**：
- 代码转换准确率 ≥ 75%（逻辑等价）
- 覆盖常见场景：UI 组件、网络请求、数据存储、权限申请
- 生成代码符合 ArkTS 规范（通过 ESLint 检查）

---

#### 功能 5：CSDN 文章自动发布

**用户故事**：
> 作为技术内容创作者，我想要把常用的问答对、代码案例整理成技术文章，自动发布到 CSDN，节省整理时间。

**功能描述**：
- 用户选择已有的问答对 / 代码案例
- 系统用 MiMo 生成完整技术文章（含代码块、配图提示词）
- 自动保存到 CSDN 草稿箱（通过 CSDN API）

**输入**：
- 问答对 / 代码案例（必填）
- 文章标题（可选，MiMo 自动生成）
- 标签（可选，默认：鸿蒙, HarmonyOS, ArkTS）

**输出**：
- 完整 Markdown 文章
- CSDN 草稿 ID（用于后续编辑 / 发布）

**验收标准**：
- 文章质量得分 ≥ 7/10（人工评估：准确性、可读性、实用性）
- 自动发布成功率 ≥ 95%
- 支持批量生成（一次生成多篇文章）

## 三、非功能需求

### 3.1 性能需求

| 指标 | 目标值 | 测量方法 |
|------|--------|---------|
| API 响应时间（P95） | < 10 秒 | 压力测试（100 并发） |
| 系统可用性 | ≥ 99.5% | 监控告警（剔除计划维护时间） |
| 并发用户数 | 支持 100 人在线 | 压力测试 |

### 3.2 安全需求

| 需求 | 描述 |
|------|------|
| API 密钥管理 | MiMo API Key 加密存储，不明文暴露在客户端 |
| 用户数据隔离 | 不同用户的对话历史、代码互不可见 |
| 输入校验 | 防止 Prompt 注入攻击（过滤恶意输入） |
| HTTPS | 全站 HTTPS（TLS 1.3） |

### 3.3 可用性需求

| 需求 | 描述 |
|------|------|
| 浏览器兼容 | Chrome 90+, Firefox 88+, Safari 14+ |
| 移动端适配 | Web UI 支持响应式布局（手机、平板可用） |
| 离线降级 | 网络异常时，提示用户检查连接（不崩溃） |

### 3.4 可扩展性需求

- 支持水平扩展（增加服务器实例应对高并发）
- 支持多模型切换（未来可接入 DeepSeek、GPT-4 等）
- 支持插件机制（第三方开发者可扩展功能）

## 四、MiMo API 使用场景（详细）

### 4.1 场景 1：RAG 问答

**调用流程**：
1. 用户提问 → MiMo Embedding API（文本向量化）
2. 向量检索（Milvus）→ Top-5 相关文档片段
3. 拼装 Prompt → MiMo Chat API（生成答案）
4. 流式返回答案（SSE）

**Token 消耗**：
- Embedding：每次 ~500 Token（提问 + 上下文）
- Chat：每次 ~1500-4000 Token（Prompt + 文档片段 + 回答）

**预计调用量**：
- 日均 50 次问答
- 月消耗：50 × 30 × 3000 = **450 万 Token**

---

### 4.2 场景 2：代码生成

**调用流程**：
1. 用户输入需求 → MiMo Chat API（生成代码框架）
2. 可选：MiMo Chat API（生成单元测试）
3. 返回完整代码（.ets 文件）

**Token 消耗**：
- 简单需求（< 50 行代码）：~3000 Token
- 中等需求（50-150 行）：~7000 Token
- 复杂需求（> 150 行）：~12000 Token

**预计调用量**：
- 日均 20 次生成
- 月消耗：20 × 30 × 6500 = **390 万 Token**

---

### 4.3 场景 3：报错诊断

**调用流程**：
1. 用户粘贴错误日志 → MiMo Chat API（分析错误）
2. 检索知识库（相似案例）→ 补充到 Prompt
3. MiMo Chat API（生成修复方案）
4. 返回 Markdown 格式诊断报告

**Token 消耗**：
- 每次 ~3000-6000 Token

**预计调用量**：
- 日均 30 次诊断
- 月消耗：30 × 30 × 4500 = **405 万 Token**

---

### 4.4 场景 4：代码迁移

**调用流程**：
1. 用户输入 Android 代码 → MiMo Chat API（理解逻辑）
2. MiMo Chat API（生成 ArkTS 实现）
3. MiMo Chat API（标注迁移注意事项）
4. 返回对比视图（Before/After）

**Token 消耗**：
- 每 100 行 Android 代码 ~10000 Token

**预计调用量**：
- 日均 10 次迁移（平均每次 80 行代码）
- 月消耗：10 × 30 × 8000 = **240 万 Token**

---

### 4.5 场景 5：CSDN 文章生成

**调用流程**：
1. 用户选择问答对 / 代码案例 → MiMo Chat API（生成文章大纲）
2. MiMo Chat API（生成完整文章）
3. 调用 CSDN API → 保存到草稿箱

**Token 消耗**：
- 每篇文章 ~5000 Token

**预计调用量**：
- 日均 5 篇文章
- 月消耗：5 × 30 × 5000 = **75 万 Token**

---

## 五、竞争分析

### 5.1 竞品对比

| 产品 | 优势 | 劣势 | 我们的差异化 |
|------|------|------|-------------|
| 华为 DevEco Studio | 官方 IDE，功能完整 | 无 AI 辅助，报错提示不友好 | AI 驱动的智能助手 |
| GitHub Copilot | 代码生成能力强 | 不支持 ArkTS，无鸿蒙专属训练 | 专注鸿蒙生态 |
| CSDN 问答社区 | 海量技术文章 | 搜索不精准，答案质量参差不齐 | RAG 精准检索 + MiMo 生成可靠答案 |
| 个人博客 | 深度实战经验 | 分散，难以系统学习 | 整合优质内容 + AI 个性化解答 |

### 5.2 竞争优势

1. **MiMo 驱动**：小米自研大模型，针对中文优化，理解 ArkTS 语法
2. **专注鸿蒙**：市面上首个专注鸿蒙开发的 AI 助手
3. **完整链路**：从问答 → 代码 → 诊断 → 迁移 → 发布，覆盖完整开发流程
4. **开源免费**：项目开源，个人开发者免费使用（依赖 MiMo Token）

| 风险 | 概率 | 影响 | 应对措施 |
